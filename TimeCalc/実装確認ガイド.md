# 実装確認ガイド（打刻方式の切り替え機能）

- 作成日: 2026-07-19
- これはローカルサーバー上で今回の実装を確認するための手順書です。

---

## 1. アクセス先

ローカルサーバーは起動済みです。ブラウザで以下を開いてください。

**http://localhost:3000/login**

もし開けない場合は、サーバーが止まっている可能性があります。その場合は次のファイルをダブルクリックしてください。

```
scripts\timecalc-server.vbs
```

黒い画面は出ず、バックグラウンドで起動します（数秒待ってからブラウザを開いてください）。

---

## 2. 確認する場所

| 確認内容 | 画面 |
|---|---|
| スタッフごとの打刻方式を切り替える | 社員一覧 → 対象スタッフの「編集」→「打刻方式」欄 |
| QRコード（標準／出勤・退勤／外出・戻りの3種類） | 設定 → 部署 → 打刻QR（部署の一覧から遷移） |
| どのQRを表示するか（部署ごと） | 設定 → 部署管理の各部署 →「表示するQR」のチェックボックス |
| 実際の打刻画面 | `/clock`（ログイン後のスタッフ本人の画面） |

---

## 3. 確認の流れ（おすすめの順番）

1. 管理者でログインする
2. 適当なスタッフを1人選び、編集画面で「打刻方式」を **スキャン即打刻** に変更して保存する
3. そのスタッフの所属部署の「打刻QR」画面を開き、3種類のQRコードが表示されることを確認する
4. スマホ（または別ブラウザ）でそのスタッフとしてログインし、「出勤・退勤QR」のURLをコピーしてアクセスする
   → タップ操作なしで「出勤 ○○:○○ を記録しました」と表示されれば成功
5. 同じURLにもう一度アクセスする
   → 「打刻済みです」と表示され、二重に記録されないことを確認する
6. 「外出・戻りQR」のURLにアクセスする
   → 外出・戻りの2つのボタンだけが表示されることを確認する

より詳しいチェックリストや、各方式の説明は [docs/打刻方式ガイド.md](docs/打刻方式ガイド.md) にまとめています。

---

## 4. コードを変更した場合

このガイドの内容を試したあとにコードを修正した場合、ローカルサーバーに反映するには再ビルド・再起動が必要です（都度私に伝えていただければ対応します）。

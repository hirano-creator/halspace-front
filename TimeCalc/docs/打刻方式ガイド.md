# 打刻方式ガイド（自由打刻 / QRタップ打刻 / スキャン即打刻）

- 作成日: 2026-07-19
- 対象: TimeCalc（株式会社ヒラノ グループ勤怠管理システム）
- 関連実装: `src/lib/auth/features.ts`（clockMode）、`src/app/(app)/clock/`、`src/app/(app)/settings/qr/[departmentId]/`

---

## 1. 3つの打刻方式

スタッフごとに、従業員編集画面（`/employees/[id]/edit` の「打刻方式」欄）で切り替える。

| 方式 | 動作 | 主な用途 |
|---|---|---|
| **自由打刻**（デフォルト） | QRなしで `/clock` を開き、出勤・退勤・外出・戻りの4ボタンから選んでタップ | 事務所勤務・PCログインで済むスタッフ |
| **QRタップ打刻** | 店舗のQRを読み取らないと打刻できない。読み取り後は画面のボタンをタップして打刻 | 打刻場所を固定したいが、ボタン操作は残したいスタッフ |
| **スキャン即打刻** | 「出勤・退勤QR」を読み取った瞬間に自動で出勤/退勤が記録される（タップ不要）。外出・戻りは「外出・戻りQR」を読み取ってからボタンをタップ | レジ横などにQRを常設し、最速で打刻を済ませたいスタッフ |

「自由打刻」以外は、店舗に掲示されたQRコード経由でのみ打刻できる（ログイン後の `/clock` に直接アクセスしても、QRコードを読み取るよう案内が出るだけでボタンは表示されない）。

---

## 2. QRコードの場所

**設定 → 部署 → 打刻QR**（URL: `/settings/qr/[部署ID]`、管理者のみアクセス可）

1部署につき QR コードが3種類表示される。

| QRラベル | 内容 | 使うスタッフ |
|---|---|---|
| 標準QR | 読み取ると出勤・退勤・外出・戻りの4ボタンが出る（従来通り） | 自由打刻／QRタップ打刻 |
| 出勤・退勤QR | 読み取った瞬間に出勤/退勤が自動記録される | スキャン即打刻 |
| 外出・戻りQR | 読み取ると外出/戻りボタンが出る（タップで打刻） | スキャン即打刻（外出・戻り時） |

- 部署の「日替わりQR」設定がONの場合、QRコードは日付が変わると自動的に無効になる（印刷ではなく常設タブレット/モニタでの表示を推奨）。**スキャン即打刻のスタッフがいる店舗は、印刷QRの使い回しを防ぐため日替わりQRの併用を推奨**（QR画面内に注記あり）。
- 「出勤・退勤QR」「外出・戻りQR」は既存の「標準QR」と共存する。標準QRを印刷済みの店舗も、そのまま使い続けて問題ない。
- **部署ごとに、どのQRを表示するか選べる**（設定 → 部署管理の各部署に「表示するQR」欄。標準QR／出勤・退勤QR／外出・戻りQRをチェックボックスで個別にON/OFF）。使わない種類のQRを非表示にして、その店舗に不要なQRを画面・印刷物に出さないようにできる。全部OFFにすると、QR表示画面に案内メッセージだけが出る。

## 3. スキャン即打刻の詳しい動き

- **出勤・退勤QRを読む** → その場で状態を判定し、未出勤なら「出勤」、出勤中なら「退勤」を自動記録する。ボタン操作は不要。
- **直近2分以内に打刻済みの場合は自動記録しない**（誤操作・リロードによる二重打刻の防止）。「〇〇 HH:mm を打刻済みです」と表示され、それでも打刻し直したい場合のみ「続けて打刻する」ボタンで手動実行できる。
- **外出中に出勤・退勤QRを読んだ場合**は、自動記録せず「外出中です。退勤しますか?」の確認ボタンを表示する（外出のまま誤って退勤扱いになるのを防ぐ）。
- **外出・戻りQRを読んだ場合**は自動記録しない。外出/戻りボタンをタップして打刻する（必ずQRのある場所に行く必要がある）。

---

## 4. 設定手順

1. `/settings/qr/[部署ID]` で「出勤・退勤QR」「外出・戻りQR」を印刷 or 常設ディスプレイに表示する。
2. `/employees/[id]/edit` を開き、「打刻方式」を「スキャン即打刻」に変更して保存する。
3. 対象スタッフのスマホで、実際にQRコードを読み取って動作確認する（§5参照）。

※ QRを設置する前に打刻方式だけ切り替えると、そのスタッフは打刻方法が無くなる（案内メッセージのみ表示）ので、**QR設置 → 方式変更の順で行うこと**。

## 5. 動作確認チェックリスト

実機（スマホ）での確認手順。

- [ ] 対象スタッフでログインし、素の `/clock`（QRなし）にアクセス → ボタンが無く、QRコードを読み取るよう案内が出る
- [ ] 「出勤・退勤QR」を読み取る → タップなしで「出勤 HH:mm を記録しました」が表示される
- [ ] 直後にもう一度同じQRを読み取る → 「打刻済みです」と表示され、二重に記録されない
- [ ] 「外出・戻りQR」を読み取る → 外出・戻りボタンのみ表示される（出勤・退勤ボタンは出ない）
- [ ] 外出ボタンをタップ → 記録される。戻りボタンも同様に確認
- [ ] （管理者側）`/attendance` または対象スタッフの詳細ページで、記録した打刻が反映されていることを確認

## 6. 制約・注意点

- QRコードの種類（出勤・退勤／外出・戻り）と打刻種別の整合はサーバー側でも検証している。QRを介さない不正な打刻リクエストは拒否される。
- GPS判定（部署に位置情報を設定している場合）は打刻方式によらず従来通り適用される。スタッフ個別に無効化したい場合は「打刻時にGPS位置情報チェックを行う」のチェックを外す（従業員編集画面）。
- 打刻方式を「自由打刻」に戻すと、QRなしの従来の4ボタン画面に即座に戻る（データや設定の追加削除は不要）。

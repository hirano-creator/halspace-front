## 1. 非エンジニアに最強の使い方：エラーをそのまま貼り付ける

Claude Codeを使い始めて気づいた、最も価値ある使い方が「**エラーをそのまま貼り付けて解決してもらうこと**」です。

プログラムを動かしていると、必ずエラーが出ます。初心者の頃は「何が書いてあるのか全くわからない」「どうやって調べればいいかもわからない」という状態になりがちです。

Claude Codeなら、エラーメッセージをそのまま貼り付けるだけで解決してくれます。

### 実際の使い方

Pythonのコードを実行してエラーが出たとします。

```
Traceback (most recent call last):
  File "app.py", line 15, in <module>
    result = total / count
ZeroDivisionError: division by zero
```

このエラーをそのままClaude Codeに渡すだけでOKです。

**入力する指示の例：**
```
app.pyを実行したらこのエラーが出ました。原因と直し方を教えて、そのまま直してください。

Traceback (most recent call last):
  File "app.py", line 15, in <module>
    result = total / count
ZeroDivisionError: division by zero
```

Claude Codeはエラーの原因を説明しながら、自動的にファイルを修正してくれます。エラーの意味を自分で理解する必要はありません。

> 📸 **[画像1]** エラーメッセージをClaude Codeに貼り付けた画面と、Claude Codeが原因を説明しながらファイルを修正する提案をしている画面。

## 3. 私が実際にClaude Codeで作ったもの

正直に言います。**Claude Codeなしでは、これらを自分一人では作れませんでした。**

### 作例①：おみくじWebアプリ

> 📸 **[画像2]** おみくじアプリの完成画面。ボタンを押すとおみくじの結果が表示される。

「ボタンを押すとおみくじの結果がランダムに出るシンプルなWebページ」を作りました。大吉・中吉・小吉・凶がランダムに出て、それぞれ違うメッセージが表示されます。

**最初の指示：**
```
ボタンを押すと「大吉・中吉・小吉・凶」がランダムで表示されるおみくじWebページを作ってください。
結果に応じてメッセージも変わるようにしてください。
ファイルはHTML一枚で完結させてください。
```

**作業にかかった時間：** 約20分

**費用：** 数円程度

最初の提案から修正まで全部で5〜6回やりとりして完成しました。「デザインをもっと和風にして」「フォントを変えて」「アニメーションを追加して」という追加指示を出しながら仕上げました。

### 作例②：鋼材計算ツール

> 📸 **[画像3]** 鋼材の重量を自動計算するWebアプリの画面。寸法を入力すると重量が自動計算される。

本業で鋼材を扱うことがあり、「サイズを入れると重量が計算されるツールが欲しい」と思って作りました。型番・寸法・本数を入力すると、合計重量と金額の目安が出るWebアプリです。

**最初の指示：**
```

スマートフォンでも使えるデザインにしてください。
```

**作業にかかった時間：** 約1時間（修正を含む）

このツールは今でも実際に使っています。以前はExcelで計算していたので、明らかに作業が速くなりました。

## 4. 「うまくいかない」を防ぐ指示のコツ

使っていて学んだ「こう頼むと失敗が少ない」というコツをまとめます。

### コツ①：小さく分けて頼む

「全部一度に作って」より「まずこれだけ作って」のほうが成功率が上がります。

**やりがちな頼み方（失敗しやすい）：**
```
ログイン機能付きのタスク管理アプリを作ってください
```

**うまくいく頼み方：**
```
まず、タスクを追加・削除できるだけのシンプルなHTMLページを作ってください。
ログインは後で追加します。
```

### コツ②：「今の状態」を伝える

修正を頼むときは「今どうなっているか」を説明すると精度が上がります。

```
ブラウザで開いたら、文字が左端に寄って表示されています。
真ん中に来るように修正してください。
```

### コツ③：制約を先に伝える

「HTMLファイル一枚で完結させて」「外部のライブラリは使わないで」など、制約を最初に伝えると意図外の方向に進まなくなります。

```
PythonだけでCSVを処理するスクリプトを作ってください。
外部ライブラリ（pandasなど）は使わずに、標準ライブラリだけで実装してください。
```

### コツ④：実行前に確認させる

大きな変更を頼むときは、先に計画を確認する習慣をつけると安心です。

```
このフォルダにあるすべてのHTMLファイルを修正したいのですが、
実行する前に「何をどのように変えるか」を説明してください。
```

## 5. よくある失敗と対処法

使っていて実際につまずいた失敗を正直に書きます。

### 失敗①：指示が曖昧すぎて全然違うものができた

**状況：** 「いい感じのページを作って」と言ったら、全く想像と違うデザインで出てきた。

**対処：** 「いい感じ」という表現は使わない。色・フォントサイズ・レイアウト・背景色を具体的に指定する。参考にしたいサイトのURLを貼り付けて「こういう雰囲気で」と伝えるのも有効。

### 失敗②：一度に頼みすぎて途中で止まった

**状況：** 「Aも、Bも、Cも、Dも全部やって」と頼んだら、Cの途中でClaude Codeが止まってしまった。

**対処：** 大きな作業は必ず小分けにする。一度のやりとりで頼む量は「ファイル2〜3個まで」が目安。

### 失敗③：意図していないファイルが変更された

**状況：** あるファイルを直してほしかったのに、別のファイルまで変更されていた。

**対処：** 「このファイルだけ変更してください」とファイル名を明示する。また、大事なファイルはバックアップを作っておく習慣をつける。

> **Gitを使うと解決します**：プログラムのファイル管理には「Git」というツールが標準的に使われています。変更を記録しておいて、「やっぱり元に戻したい」ときに戻せる仕組みです。Claude Codeとの相性も抜群で、次のステップとして学ぶ価値があります。（入門記事：「Git入門」で検索）

### 失敗④：料金が思ったより増えていた

**状況：** 気づいたら意外と使用量が増えていた。

**対処：** 下で説明する「月の上限設定」をしておく。また、「考えさせる」系の作業（複雑な相談を長々と続ける）はAPIの使用量が増えやすい。シンプルな指示・シンプルな確認を心がける。

## 6. 料金を安心して使うための設定

**Anthropic Consoleで月の上限金額を設定する方法**を紹介します。

### 手順

1. `https://console.anthropic.com` にログイン
2. 左メニューの「Billing」→「Usage Limits」を開く
3. 「Monthly Spend Limit」に上限金額を入力（例：`10`で$10≒1,500円）
4. 「Save」をクリック

> 📸 **[画像4]** Anthropic ConsoleのBillingページ。「Monthly Spend Limit」の入力欄と「Save」ボタンが見える状態。

この設定をしておくと、上限に達した時点でそれ以上の使用ができなくなります。初めての方は**$5〜$10（800〜1,500円程度）**に設定しておくと安心です。

**最後に伝えたいこと**

私がClaude Codeを使い始めて最も変わったのは、「プログラミングへの心理的なハードル」です。以前は「エラーが出たら詰み」と思っていましたが、今は「エラーが出たらClaude Codeに聞けばいい」と思えます。これだけで、作れるものの幅が全然違います。

非エンジニアにとって、Claude Codeは「自分でコードを書けるようになるツール」ではなく、「コードが書けなくても動くものを作れるツール」です。目的は作ること。Claude Codeはその手段です。

まずは**「今日の日付を表示するHTMLを作ってみること」から始めてみてください。**それだけで、「あ、自分にもできるかも」という感覚が来るはずです。

## 1. 作業フォルダの準備

Claude Codeは、**指定したフォルダの中を作業場所として扱います**。最初にどのフォルダで作業するかを決めておく必要があります。

### 手順

1. デスクトップを右クリック→「新規作成」→「フォルダー」
2. フォルダ名を `first-project` にする（半角英数字が無難です）
3. フォルダの中には何も入れなくてOK（空っぽで大丈夫）

> 📸 **[画像1]** デスクトップに「first-project」フォルダが作成された状態。

---

## 2. Claude Codeを起動する

フォルダを作ったら、そのフォルダでClaude Codeを起動します。

### 手順

1. 作成した `first-project` フォルダを **Shift + 右クリック**
2. 「PowerShellウィンドウをここで開く」を選択

> 📸 **[画像2]** フォルダを右クリックしたメニュー。「PowerShellウィンドウをここで開く」を赤枠で強調。

3. 開いたPowerShellに以下を入力してEnterを押す

```powershell
claude
```

> 📸 **[画像3]** claudeコマンドを入力した直後のPowerShell画面。「>」のプロンプトが表示されて入力待ちになった状態。

これでClaude Codeが起動した状態になりました。この「`>`」マークのあとが、指示を入力する場所です。

---

## 3. 日本語で最初の指示を出す

Claude Codeへの指示は**完全に日本語でOK**です。英語が得意でなくても問題ありません。

今回は以下の指示をそのままコピーして貼り付けてみてください。

```
今日の日付を表示するシンプルなHTMLファイルを作ってください。
デザインはシンプルで、ページの真ん中に大きく日付が表示されるようにしてください。
ファイル名はindex.htmlにしてください。
```

Enterを押すと、Claude Codeが返答を始めます。

> 📸 **[画像4]** 上記の指示をPowerShellに入力してEnterを押した直後の画面。Claude Codeが返答を生成し始めているところ。

---

## 4. Claude Codeの返答を読む

Claude Codeの返答は英語で始まることもありますが、指示は日本語で理解しています。返ってくる内容は大体このような構成です。

**返答の構成（典型例）：**

```
I'll create a simple HTML file that displays today's date in the center of the page.

● Create index.html

  <!DOCTYPE html>
  <html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>今日の日付</title>
    ... （コードが続く）
  </head>
  ...

Do you want to proceed? (y/n)
```

これが何を言っているかというと：

- 「`index.html`というファイルを作ります」という提案
- 作成予定のコードの中身（見なくてOK）
- **「実行しますか？（y/n）」という確認**

> 📸 **[画像5]** Claude Codeの返答画面全体。コードの提案と「Do you want to proceed?」の質問が見える状態。赤枠で「Do you want to proceed?」を強調。

---

## 5. 「y/n」の選択——承認って何をすること？

Claude Codeは、**何かをする前に必ず確認してきます**。これはとても大事な設計で、「勝手にファイルが書き換えられた」という事故を防いでくれます。

「y」を入力してEnterを押す → 提案した変更を実行する（ファイルを作る・書き換えるなど）
「n」を入力してEnterを押す → キャンセルする

今回は「y」を入力してEnterを押してください。

> 📸 **[画像6]** 「y」を入力してEnterを押した後の画面。「Created index.html」のような完了メッセージが表示されている。

---

## 6. ファイルができているか確認する

「y」を押した後、`first-project` フォルダの中を確認してみてください。

> 📸 **[画像7]** エクスプローラーで first-project フォルダを開いた画面。「index.html」ファイルが存在している。

`index.html` というファイルができていれば成功です！Claude Codeが自動的にファイルを作ってくれました。

`index.html` をダブルクリックすると、ブラウザで開きます。

今日の日付が大きく表示されていれば、作業成功です！コードを一行も自分で書かずに、動くWebページができました。

## 8. うまく伝わらなかったときの修正の仕方

実際に使っていると「思った通りにできた！」だけでなく「ちょっと違う…」というケースもあります。そのときはどうするか。

### ポイント：追加の指示を続けて出せばいい

Claude Codeのいいところは、**会話を続けながら修正できること**です。一度で完璧に作る必要はありません。

**例：日付の表示色が気に入らなかった場合**
```
日付の文字を青色にしてください
```

**例：フォントを変えたい場合**
```
フォントをより丸みのあるデザインにして、フォントサイズも少し大きくしてください
```

**例：全体的に修正してほしい場合**
```
今の画面を見ました。日付がもう少し下に来るようにして、背景色も白ではなくライトグレーにしてほしいです
```

このように、**「今どんな状態か」「どう変えてほしいか」を日本語で伝えるだけ**でOKです。

### 伝え方のコツ：「悪い例」と「良い例」の比較

| 伝わりにくい指示 | 伝わりやすい指示 |
|---------------|---------------|
| 「もっとよくして」 | 「フォントを大きくして、余白も広げてください」 |
| 「うまくいきません」 | 「ブラウザで開いたら真っ白な画面が表示されました」 |
| 「シンプルにして」 | 「装飾をなくして、背景を白、文字を黒にしてください」 |
| 「全部直して」 | 「レイアウトは今のままで、色だけ変えてください」 |

**具体的なほど、意図通りに動きます。** 最初は「こんなに細かく書かなきゃいけないの？」と思うかもしれませんが、慣れると自然と具体的な指示が書けるようになります。

### 実行前に確認させる方法

「いきなり変更されるのが怖い」という場合は、最初にこう付け加えると安心です。

```
実行する前に、何をするつもりか教えてください
```

Claude Codeは変更内容を先に説明してから確認を求めてくれます。

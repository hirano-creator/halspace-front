# 「What'sNoに直接保存」機能 — 計画書

## 1. 背景・目的

現状、What'sNoへファイルを入れる手段は **ダッシュボードのアップロードモーダル（ファイル選択 / ドラッグ＆ドロップ）** のみ。
このため次の不便がある。

- **メールの添付ファイル**を保存したい → 一度ローカルにダウンロード → ダッシュボードを開く → アップロード、という3手間。スマホだと特に面倒。
- **他アプリで作った資料**（Word / Excel / PDF / 写真など）を保存したい → 同じく一度ダウンロードしてから手動アップロードが必要。

ユーザーの要望は「メール添付や作成資料を**直接** What'sNo に保存したい」。
これを **両ニーズ（添付・作成資料）を1つの導線で / スマホもPCも / インフラ追加なしの軽量実装** で実現する。

## 2. 方針（決定事項）

ヒアリング結果に基づき以下を採用：

| 観点 | 決定 |
|------|------|
| 解決範囲 | メール添付・作成資料の**両方を一つの導線**で |
| 対象端末 | スマホ・PC**両対応** |
| 実装の重さ | **まず軽量に**（既存アップロードAPIを流用、メール受信基盤など新インフラは追加しない） |

### 中心アイデア：ユニバーサル「保存」受け皿ページ ＋ PWA共有先

1. **PWA Web Share Target（manifest.json）**
   既にPWA基盤（`manifest.json` / `sw.js` / `pwa.js`）があるので、`share_target` を追加するだけで、
   **Android のOS共有メニュー**（メールアプリ・写真・ファイル等の「共有」ボタン）に **「What'sNo」** が出るようになる。
   ユーザーは添付や資料を選んで「共有 → What'sNo」でそのまま保存できる。

2. **受け皿ページ `app/save.html`（新規）**
   共有されたファイルを受け取って保存する画面。同時に、共有以外（PC・iOS）からも使える
   **ドラッグ＆ドロップ ＋ ファイル選択** の保存窓口を兼ねる。1ページで全導線をカバーする。

3. **Service Worker（sw.js）で共有POSTを傍受**
   Web Share Target はファイルを `POST` で送ってくる。これを `sw.js` が受け取り、
   ファイルを一時キャッシュに退避 → `save.html` へリダイレクトして渡す（標準的なPWA共有レシピ）。

### 端末ごとの効き方

| 端末 | 導線 | 効果 |
|------|------|------|
| **Android（PWAインストール済）** | OS共有メニュー → What'sNo | ◎ メール添付・写真・資料をワンタップ保存（最も「直接」） |
| **PC（ブラウザ）** | `save.html` にD&D／ファイル選択。PWAなら長押しショートカット「保存」 | ◎ メール添付や資料をブラウザにドラッグするだけ |
| **iOS** | ホーム画面アイコン → `save.html` でファイル選択／写真選択 | ○ 共有シートへの登録はiOS制約で限定的。手動取り込みで対応（v1）|

> iOSのOS共有シートへの完全統合は Safari の制約で不可。確実にやるなら「メール転送アドレス方式（受信基盤追加）」が必要だが、今回は**軽量方針**のためv2以降の課題とする（→ ロードマップ参照）。

## 3. スコープ

### やること（v1）
- `manifest.json` に `share_target` と `shortcuts`（長押しショートカット「保存」）を追加
- `app/save.html` を新規作成（共有受け取り＋D&D＋ファイル選択、進捗表示、保存後の導線）
- `sw.js` に共有POST傍受＋一時キャッシュ退避を追加（キャッシュ版数bump）
- ダッシュボードから `save.html` への入口リンクを1か所追加（任意・低リスク）

### やらないこと（v1スコープ外）
- メール受信基盤（転送アドレス方式）→ v2候補
- iOS共有シートへの完全統合 → OS制約。Shortcutsアプリ連携はv2候補
- 保存先フォルダ/プロジェクト選択 → 現状フラット保存のまま（既存仕様踏襲）
- バックエンド（solid-api）の変更 → **不要**（既存 `POST /api/wn/files` をそのまま使う）

## 4. 影響範囲

| ファイル | 変更内容 | リスク |
|----------|----------|--------|
| `manifest.json` | `share_target`・`shortcuts` 追加 | 低（既存キー不変更） |
| `sw.js` | POST傍受分岐追加・`CACHE_NAME` bump・shellに `save.html` 追加・共有キャッシュ保護 | 中（SWは全体に効くため要検証） |
| `app/save.html`（新規） | 受け皿ページ | 低（独立ページ） |
| `app/dashboard.html` | `save.html` への入口リンク1か所（任意） | 低 |

- **バックエンド変更なし**（`wnUploadFile()` → `POST /api/wn/files` を流用）。
- **DBスキーマ変更なし**。
- 既存アップロードモーダルは**そのまま残す**（後方互換）。

## 5. 検証計画

1. **ローカル静的配信**で `save.html` を開き、D&D／ファイル選択 → `wnUploadFile()` が呼ばれ進捗が出ることを確認（API はモック/ローカル）。
2. **SWの共有傍受**：DevTools の Application → Service Workers でSW更新、`save.html` への疑似POST（fetchでmultipart送信）→ 303リダイレクト → ファイルが復元されることを確認。
3. **Android実機**：本番（Cloudflare Pages）へpush後、PWAをインストール → メールアプリの添付を「共有 → What'sNo」→ `save.html` で受信・保存できることを確認。
4. **長押しショートカット**：PWAアイコン長押しに「保存」が出ること。
5. **回帰**：既存ダッシュボードのアップロード／一覧／オフライン表示がSW変更後も正常なこと（`CACHE_NAME` bump によるキャッシュ更新含む）。

## 6. ロールアウト

- What'sNo は push = Cloudflare Pages 即時反映（[[whatsno_deploy]] 参照）。
- SW更新は `pwa.js` の更新バナー（「アップデートがあります」）で利用者へ通知される。
- アイコン素材は既存（`icon-192.png` 等）を流用。

## 7. 今後（v2以降の候補）

- **メール転送アドレス方式**：`save+xxxx@in.whatsno...` に転送→添付を自動保存（Mailgun/CloudMailin等のInbound受信基盤が必要・有料）。iOSでも確実。
- **iOS Shortcuts連携**：共有シートにショートカットを登録し `save.html` へPOST。
- **保存先フォルダ/タグの事前指定**、共有時のAIタグ自動付与。
- **スキル連携**：保存直後に「これ◯◯さんに送って」等スキルバーへ橋渡し。

### ニュース自動取り込み（Railway専用サービス・常駐ループ方式）
`aa:import-news`（RSS取得→重複排除→`aa_posts(kind=news)`保存）は、Web本体（`halspace-api`）とは別の
Railwayサービス **`aa-news-cron`** で **毎日06:00 / 12:00 / 17:00 JST に各3件まで** 自動実行される
（1回に全件流さず時間帯を分けて小出しにする。取り込みきれなかった分は次回以降に持ち越し）。

RailwayダッシュボードのCron Schedule機能（ダッシュボード操作必須）は使わず、
`docker/entrypoint.sh` に自前実装した常駐ループ方式を採用：`CRON_COMMAND` と `CRON_DAILY_UTC`
（カンマ区切りで複数時刻可。例 `21:00,03:00,08:00`）が設定されたサービスは、Webサーバーを
起動する代わりに「直近のUTC時刻までsleepしてコマンド実行、を無限に繰り返す」プロセスとして
起動し続ける。**設定・停止・再開の全操作がRailway CLIだけで完結**し、ダッシュボードでの追加設定は不要。

**環境変数（`aa-news-cron`サービス）**
| 変数 | 値 | 用途 |
|---|---|---|
| `CRON_COMMAND` | `php artisan aa:import-news --limit=3` | 実行するArtisanコマンド（`--limit=N`で1回の新規投稿数を制限。フィード間で交互に選びカテゴリの偏りを防ぐ） |
| `CRON_DAILY_UTC` | `21:00,03:00,08:00` | 毎日これらの時刻(UTC)に実行（JST 06:00 / 12:00 / 17:00 相当） |
| `CRON_ENABLED` | `true` / `false` | `false`でループは止めず実行だけスキップ（一時停止） |
| `APP_KEY`/`APP_ENV`/`APP_DEBUG`/`DB_*`/`LOG_*`/`GEMINI_API_KEY` | `${{halspace-api.VAR}}` | `halspace-api`からのクロス参照。値はコピーせず自動追従 |

**一時停止・再開（CLIのみ・ダッシュボード不要）**
```bash
# 停止（次回以降の実行をスキップ。ループ自体は継続、翌日以降いつでも再開可）
railway variable set CRON_ENABLED=false --service aa-news-cron

# 再開
railway variable set CRON_ENABLED=true --service aa-news-cron

# 現在の設定値を確認
railway variable list --service aa-news-cron --kv | grep CRON_
```
変数変更は自動的にサービスを再デプロイし、コンテナ再起動時に新しい値を読み込む
（再起動しても次回実行時刻の計算はやり直されるだけで、既存の待機予定に影響はない）。

**動作確認**
```bash
railway logs --service aa-news-cron --deployment --lines 50
```
起動直後は `[cron] daemon mode: daily at 21:00,03:00,08:00 UTC ...` と `[cron] next run: ... UTC (sleep Ns)` が出る。
手動で即時1回実行したい場合（本番DBに書き込まれる点に注意）：
`railway run --service aa-news-cron -- php artisan aa:import-news --limit=3`

## 2. フロント（a.a / Cloudflare Pages）

**本番URL: `https://aa-sns.pages.dev`**
- Cloudflare Pages プロジェクト名: `aa-sns`
- GitHub連携: `hirano-creator/halspace-front` リポジトリ、root directory = `a.a`、branch = `main`
- push → 自動デプロイ（ビルドなし・静的）

`assets/aa-api.js` がホスト名で API URL を自動判定（本番→Railway）。

### 追加デプロイが必要な場合（手動）
```bash
cd c:/dev/my-programming/a.a
npx wrangler pages deploy . --project-name aa-sns
```

### 公開後チェック
- `https://aa-sns.pages.dev` でログイン → フィード表示。
- DevTools→Application→Manifest でインストール可能・アイコン表示。
- スマホ実機で「ホーム画面に追加」→ アイコン起動 → ログイン。

## 3. 残課題（デプロイ前に判断）
- **公開範囲の認可**：MVPは「ログイン済み＝業界メンバー」。招待コード実装は後続。
- **SSO**：現状 a.a は `/auth/login`（email+pass）でトークン取得。What'sNoの `space_token` 共有による無ログイン遷移は将来対応可。
- **R2 CORS**：本番は署名URL方式。Pagesドメインを R2 CORS 許可に追加が必要な場合あり。
- **`mock/`・`docs/` を本番公開しない**こと。

# a.a デプロイ手順（Runbook）

> 構成：フロント=Cloudflare Pages（静的）／API=Railway（Laravel・halspace-api）。デプロイ順は **API→フロント**。

## ⚠ 重要：APIは共有
`halspace-api` リポジトリは **What'sNo と同じ本番API**。main への push は**What'sNoにも影響する本番デプロイ**。必ずローカル検証後・承認の上で行う。

### マイグレーション（本番DB）
Railwayが起動時に `migrate` を実行する設定ならpushで自動適用。されない場合は手動：
```bash
railway run -- php artisan migrate --path=database/migrations/2026_06_20_000001_create_aa_tables.php --force
```
（item_masters不整合があるため全体migrateでなく --path 個別適用が安全）

### ニュース自動取り込み（Railway Cronサービス）
`aa:import-news`（RSS取得→重複排除→`aa_posts(kind=news)`保存）は、Web本体（`halspace-api`）とは別に
Railwayの専用サービス **`aa-news-cron`** から **毎日06:00 JST（UTC 21:00）** に自動実行される。
`docker/entrypoint.sh` の `CRON_COMMAND` 分岐（`CRON_COMMAND="php artisan aa:import-news"`が設定されたサービスは
そのコマンドを1回実行して終了・Webサーバーは起動しない）を利用する設計。

**構築済み（CLIで実施済み）**
- サービス`aa-news-cron`作成（`hirano-creator/halspace-api`リポジトリ, branch=main）
- 環境変数：`CRON_COMMAND`のほか、`APP_KEY`/`APP_ENV`/`APP_DEBUG`/`DB_*`/`LOG_*`/`GEMINI_API_KEY`を
  `halspace-api`サービスからのクロス参照（`${{halspace-api.VAR}}`）で設定。値そのものはコピーしないため
  `halspace-api`側の値を変更するだけで自動的に反映される。

**ダッシュボードでの手動設定が必要（CLI非対応のため）**
Railwayダッシュボード → `poetic-intuition`プロジェクト → `aa-news-cron`サービス → Settings で：
1. **Source（Config-as-code Path）** を `railway.cron.json` に変更（既定の`railway.json`＝Web用のままだと`cronSchedule`が効かない）
2. **Deploy ▸ Cron Schedule** に `0 21 * * *` を設定（UTC基準＝JST 06:00）
3. **Deploy ▸ Restart Policy** を `Never` に設定
4. このサービスに **Domain（Generate Domain）は生成しない**（ヘルスチェック対象から外すため）

設定後、翌日06:00 JSTに自動実行される（DBの`aa_posts`の`kind=news`行の`created_at`で確認可）。
手動で即時実行したい場合：`railway run --service aa-news-cron -- php artisan aa:import-news`
（または`halspace-api`側から`railway run -- php artisan aa:import-news`でも同じDBに書き込まれる）

## 2. フロント（a.a / Cloudflare Pages）

**本番URL: `https://aa-sns.pages.dev`**
- Cloudflare Pages プロジェクト名: `aa-sns`
- GitHub連携: `hirano-creator/halspace-front` リポジトリ、root directory = `a.a`、branch = `main`
- push → 自動デプロイ（ビルドなし・静的）

`assets/aa-api.js` がホスト名で API URL を自動判定（本番→Railway）。

### 追加デプロイが必要な場合（手動）
```bash
cd c:/dev/my-programming/a.a
npx wrangler pages deploy . --project-name aa-sns
```

### 公開後チェック
- `https://aa-sns.pages.dev` でログイン → フィード表示。
- DevTools→Application→Manifest でインストール可能・アイコン表示。
- スマホ実機で「ホーム画面に追加」→ アイコン起動 → ログイン。

## 3. 残課題（デプロイ前に判断）
- **公開範囲の認可**：MVPは「ログイン済み＝業界メンバー」。招待コード実装は後続。
- **SSO**：現状 a.a は `/auth/login`（email+pass）でトークン取得。What'sNoの `space_token` 共有による無ログイン遷移は将来対応可。
- **R2 CORS**：本番は署名URL方式。Pagesドメインを R2 CORS 許可に追加が必要な場合あり。
- **`mock/`・`docs/` を本番公開しない**こと。

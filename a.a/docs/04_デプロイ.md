# a.a デプロイ手順（Runbook）

> 構成：フロント=Cloudflare Pages（静的）／API=Railway（Laravel・halspace-api）。デプロイ順は **API→フロント**。

## ⚠ 重要：APIは共有
`halspace-api` リポジトリは **What'sNo と同じ本番API**。main への push は**What'sNoにも影響する本番デプロイ**。必ずローカル検証後・承認の上で行う。

---

## 1. API（halspace-api / Railway）
ローカル `c:\laragon\www\solid-api`（remote=origin=hirano-creator/halspace-api、branch=main）。

### コミット対象（aa_* 一式のみ。WIPや無関係ファイルを混ぜない）
- 新規：`database/migrations/2026_06_20_000001_create_aa_tables.php`
- 新規：`app/Models/Aa/*`、`app/Http/Controllers/Aa/*`、`app/Services/AaNotificationService.php`
- 新規：`app/Console/Commands/AaImportNews.php`、`config/aa.php`
- 編集：`routes/api.php`（/aa ルート追加）、`app/Console/Kernel.php`（ニュース毎時）

```bash
cd c:/laragon/www/solid-api
git add database/migrations/2026_06_20_000001_create_aa_tables.php \
        app/Models/Aa app/Http/Controllers/Aa app/Services/AaNotificationService.php \
        app/Console/Commands/AaImportNews.php config/aa.php \
        routes/api.php app/Console/Kernel.php
git diff --staged   # ← aa_* 以外が混じっていないか必ず確認
git commit -m "feat(aa): 経営者向け業界SNS a.a の公開レイヤ(aa_*)・API・自動ニュースを追加"
git push origin main
```

### マイグレーション（本番DB）
Railwayが起動時に `migrate` を実行する設定ならpushで自動適用。されない場合は手動：
```bash
railway run -- php artisan migrate --path=database/migrations/2026_06_20_000001_create_aa_tables.php --force
```
（item_masters不整合があるため全体migrateでなく --path 個別適用が安全）

### ニュース取り込み（任意）
スケジューラ稼働後は毎時自動。手動投入：`railway run -- php artisan aa:import-news`

## 2. フロント（a.a / Cloudflare Pages）
`c:\dev\my-programming\a.a` を Pages に公開。APIベースは `assets/aa-api.js` がホスト名で自動判定（本番→Railway）。

### 方法A：Wrangler で直接デプロイ
```bash
cd c:/dev/my-programming/a.a
npx wrangler pages deploy . --project-name aa
```
- `mock/`・`docs/` は公開不要 → デプロイ前に除外するか別ディレクトリへ。
- 初回は Cloudflare ログイン（`npx wrangler login`）。

### 方法B：GitHubリポジトリ連携（推奨・push即時反映）
1. a.a を GitHub リポジトリ化（`mock/`・`docs/`・`.claude/` は `.gitignore`）。
2. Cloudflare Pages で当該リポジトリを接続。ビルド=なし（静的）、出力=ルート。
3. 以後 push で自動反映（What'sNoと同じ運用）。

### 公開後チェック
- `https://<pages-domain>/index.html` でログイン → フィード表示。
- DevTools→Application→Manifest でインストール可能・アイコン表示。
- スマホ実機で「ホーム画面に追加」→ アイコン起動 → ログイン。

## 3. 残課題（デプロイ前に判断）
- **公開範囲の認可**：MVPは「ログイン済み＝業界メンバー」。招待コード実装は後続。
- **SSO**：現状 a.a は `/auth/login`（email+pass）でトークン取得。What'sNoの `space_token` 共有による無ログイン遷移は将来対応可。
- **R2 CORS**：本番は署名URL方式。Pagesドメインを R2 CORS 許可に追加が必要な場合あり。
- **`mock/`・`docs/` を本番公開しない**こと。

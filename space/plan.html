プロジェクト概要

HaLSpaceが提供するWhat'sNoを「製造業の現場と管理側の両方が使いたくなるクラウドストレージ」として全面刷新する。


現行版はスマホ非対応・基本的なファイル管理のみ。競合（Box・Fleekdrive・DirectCloud）との差別化として、 AIタグ自動付与・QRコードリンク・バージョン管理・承認ワークフローを中核機能として実装する。


対象：中堅製造業（100〜500名）。Spaceシステムの子アプリとして space/login.html の認証を共有する。

wn_files テーブル
CREATE TABLE wn_files (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    company_id      BIGINT UNSIGNED NOT NULL,
    folder_id       BIGINT UNSIGNED NULL,
    uploaded_by     BIGINT UNSIGNED NOT NULL,
    file_name       VARCHAR(255) NOT NULL,
    storage_path    VARCHAR(500) NOT NULL,
    file_size       BIGINT UNSIGNED NOT NULL,
    mime_type       VARCHAR(100),
    version         TINYINT UNSIGNED DEFAULT 1,
    parent_id       BIGINT UNSIGNED NULL,
    status          ENUM('draft','pending_approval','approved','rejected') DEFAULT 'draft',
    approved_by     BIGINT UNSIGNED NULL,
    approved_at     TIMESTAMP NULL,
    is_deleted      BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (company_id) REFERENCES companies(id),
    FOREIGN KEY (uploaded_by) REFERENCES users(id)
);

AIタグ自動付与

Claude Haiku（claude-haiku-4-5）を使用してコスト最小化。

フロー
  1. ファイル名・拡張子・ファイルサイズをLaravelに送信
  2. PDFの場合: 先頭テキスト（1000文字）を抽出して送信
  3. Claude Haikuがタグ候補（5〜10個）をJSON配列で返す
  4. フロントエンドでユーザーが確認・選択して確定
プロンプト
製造業向けファイル管理システムです。
ファイル名: "{file_name}"
以下カテゴリからタグを最大8つ提案（JSON配列で返す）:
部門: [設計, 製造, 品管, 調達, 現場, 営業]
種別: [図面, 仕様書, 手順書, 検査表, 見積, 報告書]
工程: [組立, 加工, 検査, 塗装, 溶接, 出荷]
素材: [鉄, アルミ, ステンレス, 樹脂, 銅]
その他: [重要, 要承認, 最新版, 廃版]

  1. プレビュー（viewer.html流用）
  2. 印刷・ダウンロード
  3. 管理者画面（監査ログ・ユーザー管理）
検証手順
  1. ファイルをドラッグ&ドロップでアップロード → AIタグ提案が表示されること
  2. 同名ファイルを再アップロード → バージョンv2として保存されること
  3. QR発行 → スマホでスキャン → ファイル詳細が開くこと
  4. 承認申請 → 管理者が承認 → ステータスが「承認済み」になること
  5. スマホ（375px幅）でダッシュボードを開く → ボトムナビが表示されること
  6. タグで絞り込み → 該当ファイルのみ表示されること

ステータスフロー
draft → submitted → in_progress → review_pending
                                          ↓              ↓
                                   revision_requested  approved → delivered

形式ライブラリ対応

ファイル単位承認フロー（次の作業）
  1. モデラーが3Dファイルをアップロードして「完成・提出」→ review_pending
  2. 管理者が各ファイルに OK / 修正依頼 を設定
  3. 「検査完了・発注者へ公開」ボタンを押す
    — revision ファイルあり → revision_requested
    — 全部 ok → approved
  4. 発注者は ok ファイルのみ閲覧可能
  5. モデラーが再提出 → review_status が全件 pending にリセット
